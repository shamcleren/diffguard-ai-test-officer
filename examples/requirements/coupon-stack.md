# 需求：会员优惠券叠加

## 背景

结算页需要支持满减优惠券和会员折扣叠加。

## 验收标准

1. 订单原价满 100 元时，优惠码 `FULL100_MINUS20` 可抵扣 20 元。
2. VIP 用户可以在满减后继续叠加 95 折会员优惠。
3. 计算顺序必须是：先满减，再会员折扣。
4. 优惠后金额不能小于 0。
5. 结算页展示金额必须与后端订单预览接口返回金额一致。
6. 提交订单时传给支付服务的金额必须与最终订单金额一致。

## 示例

VIP 用户购买 100 元商品并使用 `FULL100_MINUS20`：

```text
finalPrice = (100 - 20) * 0.95 = 76.00
```
